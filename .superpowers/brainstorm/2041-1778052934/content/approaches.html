구현 접근 방식 선택

총 9개 항목 구현 — 어떤 순서로 진행할까요?

A

한 번에 모두 (Big Bang)

9개 항목을 단일 커밋으로 한꺼번에 구현

장점

  • 한 번에 완성
  • 일관성 유지 쉬움

단점

  • 중간 문제 시 디버깅 어려움
  • 리뷰 범위가 너무 큼
B

임팩트 순서 (Impact-first) ⭐ 추천

투자자 눈에 가장 먼저 보이는 것부터 3단계로 나눠 구현

1단계 · 핵심 시각
V3 목표 배분 비교 + ±5%p 이탈 강조
V4 종목별 기여도 → 수평 바 차트
2단계 · 데이터 신뢰
V2 뉴스 감성 배지 (Gemini 연동)
상관관계 히트맵 (V3 아래)
V4 툴팁 초과수익 추가
3단계 · 완성도
V1 오류 플래그 경고 배너
V1 ISA 비과세 한도 잔여
스켈레톤 UI 실제 적용
V5 HIGH 우선순위 정렬 보장
C

컴포넌트별 (Component-by-component)

V1 → V2 → V3 → V4 → V5 순서로 뷰별 완전 구현

장점

  • 뷰별로 완성 확인 가능
  • 코드 리뷰 명확

단점

  • 늦게 임팩트 큰 것 볼 수 있음
  • 단계별 맥락 전환 비용
