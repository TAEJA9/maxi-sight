# Maxi-Sight 대시보드 UI/UX 개선 설계

**날짜**: 2026-05-06  
**범위**: Skills-A~E 대비 미구현 항목 전체 + 투자자 관점 UX 이슈  
**구현 방식**: 임팩트 순서 3단계 (B안)

---

## 배경

Skills-A~E 기반으로 구현된 대시보드를 검토한 결과, 14개 항목은 구현됐으나 7개 미구현, 5개 부분 구현 상태. 투자자 관점에서 핵심 UX 이슈도 함께 해결한다.

### 제외 항목

- **전일 대비 변동**: 일별 스냅샷 데이터 없음 → 스킵

---

## 1단계 · 핵심 시각

### 1-1. V3 목표 배분 비교 (Skills-C §V3)

**변경**: 기존 2열 그리드 범례 → 현재 vs 목표 바 비교 범례로 교체

- 도넛 차트는 그대로 유지
- 범례 각 행: `[색상] [자산군명] [실선 현재 바 + 점선 목표 바] [현재%] [±diff 배지]`
- ±5%p 초과 이탈 행: 빨간 배경 + `⚠+N` 배지 + 행 테두리 강조
- 기본 목표 배분 고정값 (사용자 설정 전):

| 자산군 | 기본 목표 |
|--------|----------|
| EQ (국내 주식) | 30% |
| EQ-F (해외 주식) | 15% |
| ET (국내 ETF) | 10% |
| ET-F (해외 ETF) | 25% |
| BD (채권) | 10% |
| CS (현금/CMA) | 10% |
| CM / CR / SV | 0% (기본, 실제 보유 시 diff 표시) |

- 목표 배분은 `localStorage`에 저장, 미설정 시 기본값 적용
- D-02 트리거(리밸런싱 알림)와 연동: 이탈 항목이 있으면 V5 인사이트 자동 생성

### 1-2. V4 종목별 기여도 → 수평 바 차트 (Skills-C §V4 보조)

**변경**: 텍스트 순위 리스트 → 수익률 기반 수평 바 차트로 교체

- 바 길이 = 포트폴리오 내 최대 절대 수익률 종목 기준 상대 너비
- 양수(수익): 빨강 계열, 음수(손실): 파랑 계열 (국내 컨벤션)
- 바 내부에 티커 레이블 표시 (공간 없으면 바 외부 우측)
- 상위 6개 종목만 표시 (기존과 동일)
- 수익률 내림차순 정렬 유지

## 2단계 · 데이터 신뢰

### 2-1. V2 뉴스 감성 배지 (Skills-C §V2 보조, Skills-D D-09)

**변경**: 보조 지표 3개(Sortino/Beta/변동성) → 4번째 카드 "뉴스 감성" 추가

- 카드 구성:
  - 상단: "뉴스 감성" 레이블 + "Gemini" 출처 표시
  - 중앙: `긍정 / 중립 / 부정` 배지 (Skills-D D-09 감성 점수 기준)
  - 하단: 보유 종목별 개별 방향 아이콘 (↑ 긍정 / – 중립 / ↓ 부정)
- 감성 점수 기준 (Skills-D §1):
  - 0.5~1.0 → 긍정 (초록)
  - -0.5~0.5 → 중립 (회색)
  - -1.0~-0.5 → 부정 (보라)
- 데이터 흐름: `moduleD.js`의 `generateInsightsWithAI()` 응답에 `sentiment_scores` 필드 추가
  - 기존 Gemini 호출에 보유 종목별 감성 점수 JSON도 함께 반환하도록 프롬프트 확장
  - 응답 형식: `{ cards: [...], sentiment_scores: { "TSLA": 0.7, "SOL": -0.2, ... } }`
  - `App.jsx`에서 `sentimentScores` state로 관리 → `V2KPICards`에 prop으로 전달
- Gemini API 미연동 / API 키 없을 시 카드 완전 숨김 (에러 표시 없음)

### 2-2. 상관관계 히트맵 (Skills-B §8, Skills-C §2)

**신규**: V3 배분 차트 카드 아래 별도 `glass-card`로 추가

- 단위: **개별 종목이 아닌 자산군(asset_class) 단위** 집계
  - 이유: 종목 수 최대 14개 → 셀이 너무 작아 판독 불가, 자산군은 최대 9개로 고정
- 색상: 파랑(-1.0, 반대 방향) → 무채색(0, 독립) → 빨강(+1.0, 같은 방향)
- 자산군 수 < 2이면 섹션 전체 미표시
- 계산 방법: `moduleB.js`의 `timeline` 배열에서 자산군별 합산 시계열 분리 생성
  - `timeline` 각 포인트에 자산군별 수익률을 별도 키로 추가 (예: `timeline[i].EQ`, `timeline[i].ET_F`)
  - 이후 피어슨 상관계수 계산 (기존 `correlation_matrix` 계산 로직을 자산군 단위로 확장)
  - 자산군별 시계열이 30포인트 미만이면 해당 쌍 계산 생략, 셀에 `—` 표시
- 레전드: 파랑(-1.0 반대 방향) — 그라디언트 바 — 빨강(+1.0 같은 방향)

### 2-3. V4 툴팁 초과수익 추가 (Skills-C §V4)

**변경**: 기존 툴팁(포트폴리오 / KOSPI / S&P500) → 하단에 초과수익 행 추가

- 구분선 아래: `KOSPI 대비 초과수익 +N.NN%p`
- 음수일 경우: `-N.NN%p (KOSPI 미달)` 표시
- 색상: 양수(초과) = 빨강, 음수(미달) = 파랑 (대시보드 전체 컨벤션 통일)

## 3단계 · 완성도

### 3-1. V1 오류 플래그 경고 배너 (Skills-A §4, §5)

**신규**: PersonaInfoBar 컴포넌트 아래, V1 위에 삽입

- 표시 조건: `metrics.flags.length > 0`
- 배너 구성: `⚠️ N개 항목에 데이터 경고가 있어요 · [종목명] (사유) · [닫기]`
- 플래그 사유 한글 변환:
  - `MISSING_COST` → 취득단가 없음
  - `STALE_PRICE` → 시세 지연
  - `INVALID_QTY` → 수량 오류
- 닫기 클릭 시 세션 중 숨김 (`useState`), 페이지 새로고침 시 다시 표시
- 플래그 없으면 배너 미렌더링

### 3-2. V1 ISA 비과세 한도 잔여 (Skills-C §V1)

**신규**: ISA 계좌(code `01`) 아코디언 펼침 시 상단에 표시

- 구성: `비과세 잔여 [진행 바] 124만원 / 200만원`
- 계산: 비과세 한도 200만원 - 해당 ISA 계좌 내 이자·배당 수익 누계
  - 현재 데이터에 이자·배당 필드 없음 → `interest_krw` 합산, 없으면 0 처리
- 잔여 0원 이하: `⚠ 비과세 한도 초과` 배지로 전환
- ISA 계좌 없으면 미표시

### 3-3. 스켈레톤 UI 실제 적용 (Skills-C §5)

**변경**: 로딩 중 텍스트/스피너 → `SkeletonCard` 컴포넌트로 교체

- 적용 위치:
  - V5 인사이트 피드: AI 분석 대기 중 카드 3개 스켈레톤
  - V5 AI 건강도: `isLoading` 시 레이더 차트 영역 스켈레톤
- 기존 `SkeletonCard` 컴포넌트(`shared.jsx`) 그대로 사용
- `shimmer` 애니메이션 CSS 확인 후 없으면 추가

### 3-4. V5 우선순위 정렬 보장 (Skills-D §6)

**변경**: `moduleD.js` 인사이트 정렬 로직 보강

- 정렬 순서: HIGH → MEDIUM → LOW
- 동일 우선순위 내: `generated_at` 최신순
- HIGH 카드에 `pinned: true` 필드 추가 → V5 InsightCard에서 📌 핀 아이콘 표시
- 현재 `generateInsights()` 반환 배열에 sort 보장 추가

## 데이터 흐름 변경

```
portfolio.json
  └─ moduleA (정규화)
  └─ moduleB (지표 계산)
       └─ allocation → V3 목표 배분 비교 (localStorage target vs current)
       └─ contributions → V4 수평 바 차트
       └─ correlation_matrix (자산군 단위 신규 계산) → 히트맵
       └─ timeline → V4 툴팁 초과수익
  └─ moduleD (인사이트)
       └─ sort(HIGH→MEDIUM→LOW) → V5 피드
       └─ geminiClient → 뉴스 감성 점수 → V2 배지
```

## 비기능 요건

- 상관관계 계산은 `useMemo`로 캐싱 (매 렌더 재계산 방지)
- 뉴스 감성 카드: API 실패 시 조용히 숨김, 에러 바운더리 없이 처리
- ISA 한도: 계좌 코드 `01`만 해당, 다른 계좌 영향 없음
- 목표 배분 기본값은 `src/modules/moduleB.js` 또는 별도 `config.js`에 상수로 정의
