# Skills-D.md · 인사이트 트리거 규칙

Maxi-Sight 분석 파이프라인 Module D — Insight Trigger Rules


## 목적

Module B에서 계산된 지표값을 기준으로, 사용자에게 표시할 인사이트 카드와 알림을 자동으로 생성한다. 각 트리거는 조건 → 인사이트 유형 → 카드 생성 지시의 구조로 작동한다.


## 트리거 정의 구조

[트리거 ID] 조건

→ 인사이트 유형 | 카드 제목

→ Gemini에게 전달할 생성 지시

→ 우선순위 (HIGH / MEDIUM / LOW)


## 1. 위험 알림 트리거

### D-01 · MDD 임계값 초과

조건: mdd_pct > 10

→ 위험 알림 | "방어벽 점검이 필요해요"

→ MDD가 {mdd_pct}%로 임계값(10%)을 초과했다. 현재 포트폴리오 구성에서

어떤 자산군이 낙폭에 가장 크게 기여했는지 분석하고, 리스크를 줄이기 위한

구체적인 행동 1~2가지를 일상 언어로 제안하라.

→ 우선순위: HIGH

### D-05 · 연환산 변동성 과열

조건: volatility_annualized_pct > 25

→ 위험 알림 | "시장 과열 주의"

→ 포트폴리오 연환산 변동성이 {volatility_annualized_pct}%로 높은 수준이다.

시장 상황과 연결지어 변동성이 높아진 원인을 설명하고, 변동성을 낮출 수 있는

방법을 쉬운 언어로 1~2문장으로 제안하라.

→ 우선순위: HIGH

### D-09 · 뉴스 감성 경보

조건: gemini_news_sentiment_score < -0.5 (보유 종목 1개 이상)

→ 위험 알림 | "종목 온도계가 차가워졌어요"

→ Gemini의 Google Search 기능을 통해 보유 종목의 최근 24시간 뉴스를 분석하라.

부정적 이슈가 감지된 종목명과 핵심 이슈를 1~2문장으로 요약하고,

비중 조절 필요성을 친근한 어조로 제안하라. 공포를 조장하지 말 것.

→ 우선순위: MEDIUM

**감성 점수 산출 기준**

| 점수 범위 | 감성 | V5 배지 색상 |
|---|---|---|
| 0.5 ~ 1.0 | 긍정 | 초록 |
| -0.5 ~ 0.5 | 중립 | 회색 |
| -1.0 ~ -0.5 | 부정 | 보라 |

- 점수는 Gemini가 뉴스 헤드라인·본문을 분석해 -1.0 ~ 1.0 범위로 반환

- 종목별 개별 점수 계산 후, 포트폴리오 비중 가중 평균으로 전체 점수 산출


## 2. 리밸런싱 알림 트리거

### D-02 · 자산군 비중 목표 이탈

조건: |현재 자산군 비중 - 목표 자산군 비중| > 5 (단위: %p, 1개 이상의 자산군)

→ 리밸런싱 알림 | "포트폴리오 정리정돈이 필요해요"

→ {이탈한 자산군 목록}의 비중이 목표 범위를 벗어났다. 현재 비중과 목표 비중의

차이를 구체적으로 언급하고, 어떤 자산을 매수/매도해야 하는지 방향성을

쉬운 언어로 제안하라. 구체적인 금액보다 비율과 방향에 집중하라.

→ 우선순위: MEDIUM


## 3. 기회 신호 트리거

### D-03 · Sharpe Ratio 우수 유지

조건: sharpe_ratio > 1.5 AND 해당 조건 유지 기간 ≥ 90일

→ 기회 신호 | "효율 우수 포트폴리오 🏅"

→ Sharpe Ratio가 {sharpe_ratio}로 3개월 이상 1.5를 초과하고 있다. 이것이

얼마나 좋은 성과인지 일반 투자자가 이해할 수 있는 언어로 칭찬하고,

이 효율을 유지하기 위해 주의해야 할 점을 한 문장으로 덧붙여라.

→ 우선순위: LOW

### D-06 · 벤치마크 초과 수익 달성

조건: 최근 1개월 포트폴리오 수익률 > KOSPI 수익률 + 3%p

→ 기회 신호 | "시장을 이기고 있어요! 📈"

→ 최근 한 달간 KOSPI 대비 {초과수익률}%p 높은 성과를 냈다. 어떤 자산군이

이 성과에 기여했는지 간략히 언급하고, 이 흐름을 유지할 때 유의할 점을

친근한 어조로 한 문장 제안하라.

→ 우선순위: LOW


## 4. 집중 위험 트리거

### D-04 · 단일 종목 비중 과다

조건: 단일 종목 비중 > 20% (전체 포트폴리오 대비)

→ 집중 위험 | "한 바구니에 너무 많이 담겼어요"

→ {종목명}이 포트폴리오의 {비중}%를 차지하고 있다. 집중 투자의 위험성을

계란 한 바구니 비유 없이 새로운 표현으로 설명하고, 분산 투자를 위한

실질적인 방향을 1~2문장으로 제안하라.

→ 우선순위: MEDIUM


## 5. 정보 알림 트리거

### D-07 · 장기 미매매 알림

조건: 마지막 거래 이후 경과일 > 90일

→ 정보 알림 | "포트폴리오를 점검할 때가 됐어요"

→ 마지막 매매 후 {경과일}일이 지났다. 정기적인 포트폴리오 점검의 중요성을

부담스럽지 않은 어조로 언급하고, 지금 확인하면 좋을 항목 1~2가지를 제안하라.

→ 우선순위: LOW

### D-08 · 수익률 목표 달성

조건: total_return_pct ≥ 사용자 설정 목표 수익률

→ 정보 알림 | "목표 수익률에 도달했어요! 🎯"

→ 설정한 목표 수익률({목표값}%)에 도달했다. 수익 실현·리밸런싱·목표 상향 중

어떤 선택지를 고려할 수 있는지 중립적인 관점에서 설명하라.

특정 행동을 강요하지 말고 선택지를 제시하는 형태로 작성하라.

→ 우선순위: MEDIUM


## 6. 트리거 처리 규칙

### 우선순위별 표시 순서

1. **HIGH** → 피드 최상단 고정

2. **MEDIUM** → 날짜순 정렬

3. **LOW** → 날짜순 정렬, HIGH/MEDIUM 이후 배치

### 중복 방지

- 동일한 트리거 ID는 **24시간 이내 중복 생성 금지**

- D-01과 D-05가 동시에 발동하면 두 카드 모두 생성 (별개 조건)

- D-09와 D-01이 동시에 발동하면 D-01(HIGH)을 먼저 표시

- D-03(기회)과 D-04(집중 위험)가 동시에 발동하면 D-04를 우선 표시

### 조건 미충족 시

어떤 트리거 조건도 충족하지 않으면 V5 피드에 기본 메시지 표시:

"현재 포트폴리오에서 특이 신호가 감지되지 않았어요. 안정적으로 유지되고 있어요. 👍"


## 7. Gemini API 호출 지시

### 시스템 프롬프트 구성

- Skills-A.md ~ Skills-E.md 5개 모듈 전체를 시스템 프롬프트로 첨부

### 유저 메시지 구성

아래 포트폴리오 분석 결과를 기반으로, 발동된 트리거에 해당하는 인사이트 카드를 생성하라.

[포트폴리오 지표]

{Module B 출력 JSON}

[발동된 트리거]

{조건을 충족한 트리거 ID 목록과 관련 수치}

[출력 형식 - JSON 배열만 반환, 다른 텍스트 없이]

[

{

"trigger_id": "D-01",

"type": "위험 알림 | 리밸런싱 알림 | 기회 신호 | 집중 위험 | 정보 알림",

"title": "카드 제목",

"body": "본문 2~3문장. 쉬운 언어. 친근한 어조.",

"priority": "HIGH | MEDIUM | LOW"

}

]

### 출력 제약

- 각 카드 본문(`body`)은 **2~3문장, 80자 이내**

- 금융 전문 용어 사용 금지 (불가피한 경우 괄호 안에 한글 설명 병기)

- 문체: 격식체(`~습니다`) 사용 금지, 친근한 평어체(`~요`) 사용

- D-09 뉴스 분석 시: Google Search 결과 기반으로 실제 뉴스 요약 포함
